\section{Experimental Results}
\label{section:exp}

\subsection{Preliminary Encoder Ablation}
\label{section:exp:baseline}

Beyond the proposal stage, we have already implemented the complete system end to
end: (i) a fully synthetic data pipeline that builds monophonic scores with
\texttt{music21} and renders them to images with \texttt{verovio}, emitting four
parallel decoupled label streams (\textit{type} / \textit{pitch} / \textit{rhythm}
/ \textit{attribute}) so pitch and rhythm accuracy can be measured independently;
and (ii) our multi-head Vision-Encoder-Decoder, trained on $100{,}000$ deterministic
synthetic samples and evaluated on a held-out set of $1{,}000$ pre-rendered samples.

As the first milestone we run an \emph{encoder ablation} inside our own architecture,
holding the decoder, loss, and data fixed and swapping only the visual backbone: an
ImageNet-pretrained Vision Transformer (\texttt{vit}, with the \texttt{[CLS]} token
dropped) versus a from-scratch convolutional encoder (\texttt{resnet}). This isolates
how much of the task is solved by the visual representation alone, before we turn to
the external baselines (Oemer / CRNN / TrOCR) described in
Section~\ref{section:evaluation}. All runs use AdamW (\texttt{lr}\,$=5\!\times\!10^{-4}$,
weight decay $0.01$) with a cosine schedule ($1{,}000$ warmup steps), $30$ epochs,
\texttt{bf16} mixed precision, and gradient clipping at $1.0$; the decoder is a
$4$-layer, $6$-head Transformer with $d_{\text{model}}=384$.

Table~\ref{table:baseline_results} reports the results, and
Figure~\ref{fig:val_curves} shows the corresponding validation curves.

\subsection{Analysis}

The pretrained \texttt{vit} encoder solves the task almost perfectly, reaching a
validation SER of $0.0029$ with $99.85\%$ pitch and $99.78\%$ rhythm accuracy. The
from-scratch \texttt{resnet} learns rhythm only partially and \textbf{structurally
fails to learn pitch} (accuracy $\approx0\%$), with its pitch cross-entropy pinned at
the $\approx1.25$ ``NULL-only'' floor --- i.e.\ it predicts the correct token on
non-note positions but is uninformative on notes.

We attribute this to the encoder geometry rather than the decoder, loss, or data,
since the \emph{same} decoder and loss reach $99.85\%$ pitch with \texttt{vit}. A
translation-equivariant CNN that pools image height into a single token per column
discards the \emph{absolute vertical position} of a notehead --- which is exactly what
pitch is on a staff. The failure proved invariant to learning-rate sweeps, replacing
height mean-pooling with height-flatten, increasing vertical resolution, and adding a
learned vertical positional embedding. A proper fix requires a 2-D feature-token grid
with 2-D positional embeddings. We therefore adopt \texttt{vit} as the working
configuration and are currently scaling up its training; the external baselines from
Section~\ref{section:evaluation} (CRNN, TrOCR, and the Oemer pipeline) remain ongoing
work for the final report.
